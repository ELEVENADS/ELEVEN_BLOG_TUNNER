# ELEVEN Blog Tuner - 数据库设计

## 1. 数据库概述

### 1.1 设计原则

- 基于 SQLModel (Pydantic v2) ORM
- 支持 PostgreSQL (生产) / SQLite (开发)
- 软删除机制，支持数据恢复
- 统一的时间戳字段管理

### 1.2 表清单

| 表名                 | 说明      | 核心功能关联 |
| ------------------ | ------- | ------ |
| users              | 用户表     | 用户管理模块 |
| notes              | 笔记表     | 笔记管理模块 |
| note\_categories   | 笔记分类表   | 笔记管理模块 |
| tags               | 标签表     | 笔记管理模块 |
| note\_tag\_mapping | 笔记标签关联表 | 笔记管理模块 |
| styles             | 风格表     | 风格学习模块 |
| style\_features    | 风格特征表   | 风格学习模块 |
| articles           | 文章表     | 文章生成模块 |
| article\_versions  | 文章版本表   | 文章生成模块 |
| article\_outlines  | 文章大纲表   | 文章生成模块 |
| reviews            | 审核记录表   | 文章审核模块 |
| publications       | 发布记录表   | 文章发布模块 |
| tasks              | 任务表     | 任务调度   |
| knowledge\_base    | 知识库表    | 知识库管理  |
| system\_configs    | 系统配置表   | 系统配置模块 |

## 4. 命名规范

### 4.1 表名规范

- 使用复数形式: `users`, `articles`, `notes`
- 小写字母 + 下划线分隔
- 关联表命名: `{table1}_{table2}_mapping` 或 `{table1}_{table2}`

### 4.2 字段名规范

- 小写字母 + 下划线分隔
- 主键: `id`
- 外键: `{table_singular}_id`
- 时间戳: `created_at`, `updated_at`, `deleted_at`
- 状态字段: `status`
- 软删除: `deleted_at`

### 4.3 索引命名规范

- 普通索引: `idx_{table}_{column}`
- 唯一索引: `uq_{table}_{column(s)}`
- 外键索引: 自动创建
