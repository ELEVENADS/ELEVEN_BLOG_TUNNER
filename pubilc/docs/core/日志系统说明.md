# 日志系统说明

## 1. 系统概述

ELEVEN Blog Tuner 采用 loguru 库实现了结构化的日志系统，提供了统一的日志记录能力，支持多输出目标、日志轮转和保留策略。

## 2. 技术选型

- **日志库**: loguru 0.7.0+
- **输出目标**: 
  - 控制台输出（彩色格式化）
  - 文件输出（结构化）
  - 错误日志单独存储

## 3. 目录结构

```
eleven_blog_tunner/
├── utils/
│   ├── __init__.py       # 导出 logger 实例
│   └── logger.py          # 日志系统实现
├── logs/                  # 日志文件存储目录
│   ├── app_2026-04-17.log     # 应用日志
│   └── error_2026-04-17.log   # 错误日志
```

## 4. 核心功能

### 4.1 单例模式

日志系统采用单例模式实现，确保全局只有一个日志实例，避免重复初始化。

### 4.2 多输出配置

- **控制台输出**: 彩色格式化，显示时间、级别、模块、函数、行号和消息
- **应用日志文件**: 每日轮转，保留7天，记录 INFO 及以上级别
- **错误日志文件**: 每日轮转，保留14天，记录 ERROR 及以上级别

### 4.3 日志格式

**控制台格式**:
```
<green>2026-04-17 12:34:56</green> | <level>INFO    </level> | <cyan>module</cyan>:<cyan>function</cyan>:<cyan>123</cyan> - <level>message</level>
```

**文件格式**:
```
2026-04-17 12:34:56 | INFO     | module:function:123 - message
```

## 5. 使用方法

### 5.1 基本使用

```python
from eleven_blog_tunner.utils import logger

# 不同级别的日志
logger.debug("调试信息")      # 仅控制台输出
logger.info("信息")         # 控制台和应用日志
logger.warning("警告")      # 控制台和应用日志
logger.error("错误")        # 控制台、应用日志和错误日志
logger.critical("严重错误")   # 控制台、应用日志和错误日志

# 带上下文的日志
logger.info("用户登录", user_id="123", action="login")

# 异常日志
try:
    1 / 0
except Exception as e:
    logger.exception("发生异常")  # 自动记录堆栈信息
```

### 5.2 模块级使用

在各个模块中导入并使用 logger：

```python
# 在 agent 模块中
from eleven_blog_tunner.utils import logger

class BaseAgent:
    def __init__(self):
        logger.info(f"初始化 {self.__class__.__name__}")
    
    def run(self, task):
        logger.debug(f"执行任务: {task}")
        try:
            # 执行逻辑
            logger.info(f"任务执行完成: {task}")
        except Exception as e:
            logger.exception(f"任务执行失败: {task}")
```

## 6. 配置说明

### 6.1 日志目录

日志文件存储在项目根目录下的 `logs/` 文件夹中，系统会自动创建该目录。

### 6.2 轮转策略

- **应用日志**: 每日 00:00 轮转，保留7天
- **错误日志**: 每日 00:00 轮转，保留14天

### 6.3 级别说明

| 级别 | 数值 | 说明 | 输出目标 |
|------|------|------|----------|
| DEBUG | 10 | 调试信息 | 控制台 |
| INFO | 20 | 一般信息 | 控制台 + 应用日志 |
| WARNING | 30 | 警告信息 | 控制台 + 应用日志 |
| ERROR | 40 | 错误信息 | 控制台 + 应用日志 + 错误日志 |
| CRITICAL | 50 | 严重错误 | 控制台 + 应用日志 + 错误日志 |

## 7. 最佳实践

1. **使用适当的级别**: 根据消息的重要性选择合适的日志级别
2. **包含上下文信息**: 在日志中包含相关的上下文信息，便于问题定位
3. **异常处理**: 使用 `logger.exception()` 记录异常，自动包含堆栈信息
4. **避免敏感信息**: 不要在日志中记录密码、API 密钥等敏感信息
5. **日志量控制**: 避免过多的调试日志，影响系统性能

## 9. 故障排查

### 9.1 日志文件不存在

- 检查 `logs/` 目录是否存在，系统应该自动创建
- 检查文件权限是否正确

### 9.2 日志级别问题

- 确认使用了正确的日志级别
- 检查 loguru 配置是否正确

### 9.3 性能问题

- 减少 DEBUG 级别的日志输出
- 避免在高频调用的函数中使用大量日志

## 10. 总结

本日志系统基于 loguru 实现，提供了结构化、多输出、自动轮转的日志记录能力，为系统运行状态监控和问题排查提供了有力支持。通过统一的日志接口，确保了系统各模块日志风格的一致性，便于日志分析和处理。
