# ELEVEN Blog Tuner - 网关层说明

## 1. 模块概述

Gateway 层是 ELEVEN Blog Tuner 项目的控制系统核心，负责接收 API 请求、调度任务、监控系统状态，并与 Agent 系统深度绑定。作为用户与 Agent 系统的桥梁，Gateway 层确保系统能够 24 小时稳定运行，并高效处理各类任务。

### 1.1 核心职责

- **任务管理**：创建、调度、跟踪和执行各类任务
- **24 小时运行**：持续监控和处理任务，确保系统响应
- **API 处理**：接收和处理来自前端或外部系统的 API 请求
- **状态监控**：实时监控系统资源、任务进度和 Agent 状态
- **Agent 集成**：与 BossAgent 和 SystemAgent 深度绑定，协调任务执行

### 1.2 设计目标

- **高可用性**：确保系统 24 小时不间断运行
- **可扩展性**：支持多种任务类型和新增功能
- **可靠性**：完善的任务状态管理和错误处理
- **可观测性**：实时监控和告警机制
- **安全性**：严格的参数验证和权限控制

## 3. 核心组件详解

### 3.1 TaskManager - 任务管理器

任务管理器是 Gateway 层的核心组件，负责管理系统中所有任务的完整生命周期。

#### 主要功能

- **任务创建**：支持多种任务类型的创建
- **任务调度**：基于优先级的任务队列管理
- **任务执行**：异步执行任务，支持多种处理策略
- **状态追踪**：实时跟踪任务执行状态和进度
- **结果存储**：保存任务执行结果和历史记录

#### 任务类型

| 任务类型 | 说明 | 处理流程 |
| --- | --- | --- |
| article_generation | 文章生成 | 调用完整Agent链 |
| style_analysis | 风格分析 | 调用SystemAgent |
| article_review | 文章审查 | 调用ReviewAgent |
| system_status | 系统状态查询 | 调用SystemAgent |

#### 任务状态

```python
class TaskStatus(Enum):
    PENDING = "pending"      # 待处理
    RUNNING = "running"      # 执行中
    COMPLETED = "completed"  # 已完成
    FAILED = "failed"        # 失败
    CANCELLED = "cancelled"  # 已取消
```

#### 使用示例

```python
from eleven_blog_tunner.gateway import TaskManager

# 创建任务管理器
task_manager = TaskManager()
await task_manager.start()

# 创建任务
task_id = await task_manager.create_task(
    task_type="article_generation",
    input_data="写一篇关于人工智能的文章",
    user_id="user123"
)

# 获取任务状态
status = await task_manager.get_task_status(task_id)
print(f"任务状态: {status['status']}")

# 获取任务结果
result = await task_manager.get_task_result(task_id)
print(f"任务结果: {result}")

# 取消任务
success = await task_manager.cancel_task(task_id)

# 列出任务
tasks = await task_manager.list_tasks(user_id="user123")
```

### 3.2 APIHandler - API处理器

API 处理器负责接收和处理来自外部的 API 请求，提供标准化的接口给客户端。

#### 主要功能

- **请求接收**：接收 HTTP 请求并解析参数
- **参数验证**：验证请求参数的合法性和完整性
- **任务创建**：调用 TaskManager 创建任务
- **结果返回**：统一格式化返回响应
- **错误处理**：标准化的错误处理和响应

#### API 接口

| 接口 | 方法 | 说明 |
| --- | --- | --- |
| /api/v1/tasks | POST | 创建任务 |
| /api/v1/tasks/{id} | GET | 获取任务状态 |
| /api/v1/tasks/{id}/result | GET | 获取任务结果 |
| /api/v1/tasks/{id} | DELETE | 取消任务 |
| /api/v1/tasks | GET | 列出任务 |
| /api/v1/health | GET | 健康检查 |

#### 使用示例

```python
from eleven_blog_tunner.gateway import APIHandler

# 创建 API 处理器
api_handler = APIHandler()
await api_handler.start()

# 创建任务
response = await api_handler.create_task(
    task_type="article_generation",
    input_data="写一篇关于人工智能的文章",
    user_id="user123"
)

# 获取任务状态
status = await api_handler.get_task_status(response["task_id"])

# 获取任务结果
result = await api_handler.get_task_result(response["task_id"])

# 取消任务

# 列出任务
tasks = await api_handler.list_tasks(user_id="user123")

# 健康检查

```

### 3.3 StatusMonitor - 状态监控

状态监控组件负责实时监控系统状态、任务进度和资源使用情况。

#### 主要功能

- **系统状态监控**：收集系统运行信息
- **资源监控**：监控 CPU、内存、磁盘、网络等资源
- **任务统计**：统计任务数量、状态分布、类型分布
- **告警检测**：检测异常情况并生成告警
- **历史记录**：保存监控历史数据

#### 监控指标

| 类别 | 指标 | 说明 |
| --- | --- | --- |
| 系统 | hostname, platform, uptime | 系统基本信息 |
| CPU | cpu_percent | CPU 使用率 |
| 内存 | total, available, used, percent | 内存使用情况 |
| 磁盘 | total, used, free, percent | 磁盘使用情况 |
| 网络 | bytes_sent, bytes_recv | 网络流量 |
| 任务 | total, pending, running, completed, failed | 任务统计 |

#### 告警级别

| 级别 | 说明 | 触发条件 |
| --- | --- | --- |
| info | 信息 | 待处理任务过多 |
| warning | 警告 | CPU/内存使用率 > 80% |
| critical | 严重 | 磁盘使用率 > 90% |

#### 使用示例

```python
from eleven_blog_tunner.gateway import TaskManager, StatusMonitor

# 创建任务管理器和状态监控
task_manager = TaskManager()
status_monitor = StatusMonitor(task_manager)

await task_manager.start()
await status_monitor.start()

# 获取系统状态
system_status = await status_monitor.get_system_status()
print(f"系统状态: {system_status}")

# 获取任务统计
task_stats = await status_monitor.get_task_stats()
print(f"任务统计: {task_stats}")

# 获取资源使用
resource_usage = await status_monitor.get_resource_usage()
print(f"资源使用: {resource_usage}")

# 获取告警
alerts = await status_monitor.get_alerts()
print(f"告警: {alerts}")

# 清除告警
await status_monitor.clear_alerts()
```

### 3.4 Integration - 集成模块

集成模块负责与 Agent 系统深度绑定，特别是与 BossAgent 和 SystemAgent 的协作。

#### 主要功能

- **Agent 调用**：封装 Agent 调用逻辑
- **配置管理**：管理系统和风格配置
- **任务链执行**：执行完整的任务处理链
- **健康检查**：检查 Agent 可用性

#### 与 BossAgent 的绑定

- Gateway 层作为任务发起者
- 将用户请求转换为任务指令
- BossAgent 作为任务调度中心
- 实时反馈任务执行状态

#### 与 SystemAgent 的绑定

- SystemAgent 作为系统信息提供者
- Gateway 层通过其获取配置和状态
- 支持风格配置查询
- 支持系统参数获取

#### 使用示例

```python

# 执行任务链
chain_result = await integration.execute_task_chain(
    task_type="article_generation",
```

## 4. 与 Agent 的深度绑定

### 4.1 与 BossAgent 的绑定

BossAgent 是任务调度中心，Gateway 层与其深度绑定实现任务调度功能。

#### 绑定关系

```
Gateway 层                          BossAgent
    │                                   │
    ├── 创建任务 ──────────────────────→ │
    │                                   │
    │     分析任务类型                   │
    │                                   │
    │ ←─── 返回调度计划 ─────────────── │
    │                                   │
    ├── 监控执行状态 ←───────────────── │
    │                                   │
    │ ←─── 反馈执行结果 ─────────────── │
    │                                   │
    └── 返回最终结果                     │
```

#### 协作流程

1. **任务创建**：Gateway 层接收请求，创建任务
2. **任务调度**：调用 BossAgent 分析任务类型
3. **子任务分配**：BossAgent 调用其他 Agent
4. **状态反馈**：实时反馈任务执行状态
5. **结果整合**：整合各 Agent 执行结果
6. **返回结果**：Gateway 层返回最终结果

### 4.2 与 SystemAgent 的绑定

SystemAgent 是系统信息提供者，Gateway 层与其深度绑定获取系统配置和状态。

#### 绑定关系

```
Gateway 层                          SystemAgent
    │                                   │
    ├── 获取系统配置 ──────────────────→ │
    │ ←─── 返回配置信息 ─────────────── │
    │                                   │
    ├── 获取风格配置 ──────────────────→ │
    │ ←─── 返回风格信息 ─────────────── │
    │                                   │
    ├── 查询系统状态 ──────────────────→ │
    │ ←─── 返回状态信息 ─────────────── │
    │                                   │
    └── 返回查询结果                     │
```

#### 协作流程

1. **配置查询**：获取 LLM、RAG 等系统配置
2. **风格查询**：获取用户风格配置
3. **状态查询**：获取系统运行状态
4. **任务支持**：为任务执行提供配置支持

## 5. 24 小时运行机制

### 5.1 工作线程

TaskManager 内部维护一个工作线程，持续处理任务队列：

```python
async def _worker(self):
    """工作线程"""
    while self.running:
        if self.task_queue:
            task_id = self.task_queue.popleft()
            await self._process_task(task_id)
        else:
            await asyncio.sleep(1)  # 队列为空时休眠
```

### 5.2 任务处理流程

```
任务创建
    ↓
加入任务队列
    ↓
工作线程取出任务
    ↓
验证任务状态
    ↓
执行任务处理
    ↓
更新任务状态和进度
    ↓
保存任务结果
    ↓
继续处理下一任务
```

### 5.3 自动启动

当创建新任务时，如果任务管理器未启动，将自动启动：

```python
if not self.running:
    await self.start()
```

## 7. 使用方法

### 7.1 基本使用

```python
import asyncio
from eleven_blog_tunner.gateway import TaskManager, APIHandler

async def main():
    # 创建 API 处理器
    api_handler = APIHandler()
    await api_handler.start()

    # 创建文章生成任务
    response = await api_handler.create_task(
        task_type="article_generation",
        input_data="写一篇关于人工智能发展趋势的文章",
        user_id="user123"
    )

    task_id = response["task_id"]
    print(f"任务创建成功: {task_id}")

    # 轮询获取任务状态
    while True:
        status = await api_handler.get_task_status(task_id)
        print(f"任务状态: {status['data']['status']}")

        if status['data']['status'] in ['completed', 'failed']:
            break

        await asyncio.sleep(2)

    # 获取任务结果
    if status['data']['status'] == 'completed':
        result = await api_handler.get_task_result(task_id)
        print(f"任务结果: {result}")

    # 关闭
    await api_handler.stop()

# 运行
asyncio.run(main())
```

## 10. 注意事项

1. **线程安全**：多个组件可能同时访问共享数据，需使用锁机制
2. **资源清理**：任务完成后及时释放资源
3. **错误恢复**：系统重启后能够恢复未完成的任务
4. **日志记录**：关键操作需记录日志便于排查问题
5. **配置管理**：敏感配置需通过环境变量或配置文件管理
