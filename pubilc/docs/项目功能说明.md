# ELEVEN Blog Tuner - 项目功能说明

## 1. 项目概述

**项目名称**: ELEVEN Blog Tuner
**项目定位**: 个人博客 AI 智能助手
**核心价值**: 根据个人笔记学习写作风格，生成具有个人文笔特色的博客文章
**目标用户**: 博主、内容创作者、知识工作者

## 2. 系统功能架构

```
┌─────────────────────────────────────────────────────────────────┐
│                        用户交互层 (Vue3)                         │
├─────────────────────────────────────────────────────────────────┤
│                      OpenAPI 服务层 (FastAPI)                    │
├─────────────────────────────────────────────────────────────────┤
│                      Gateway 层 (控制系统核心)                    │
│  ┌──────────────┬──────────────┬──────────────┬──────────────┐  │
│  │ TaskManager │ APIHandler  │StatusMonitor │ Integration  │  │
│  │   任务管理器   │   API处理器   │   状态监控    │   集成模块    │  │
│  └──────────────┴──────────────┴──────────────┴──────────────┘  │
├──────────┬──────────┬──────────┬──────────┬──────────┬─────────┤
│  用户管理  │  笔记管理  │  风格学习  │  文章生成  │  文章发布  │ 系统配置 │
└──────────┴──────────┴──────────┴──────────┴──────────┴─────────┘
```

## 3. 核心功能模块

### 3.1 网关层模块 (Gateway)

Gateway 层是控制系统的核心，负责 24 小时运行、任务调度、状态监控和 Agent 协调。

#### 功能列表

- 任务创建与管理
- 24 小时自动运行
- 系统状态监控
- Agent 深度绑定
- 资源使用监控
- 告警管理

#### 功能详情

| 功能 | 说明 | 优先级 |
| --- | --- | --- |
| TaskManager | 任务管理器，24小时运行和自动处理任务链 | P0 |
| APIHandler | API处理器，接收和处理API请求 | P0 |
| StatusMonitor | 状态监控，监控系统状态和任务进度 | P1 |
| Integration | 集成模块，与BossAgent和SystemAgent深度绑定 | P0 |

#### 核心能力

- **24小时运行**：持续监控和处理任务，确保系统响应
- **任务管理**：支持多种任务类型的创建、调度和跟踪
- **状态监控**：实时监控系统资源、任务进度和Agent状态
- **深度绑定**：与BossAgent和SystemAgent紧密协作

### 3.2 用户管理模块

#### 功能列表

- 用户注册与登录
- 用户信息管理
- 密码修改与重置

### 3.3 笔记管理模块

#### 功能列表

- 笔记导入 (Markdown/TXT/PDF)
- 笔记列表查询
- 笔记编辑与删除
- 笔记分类与标签
- 笔记搜索

#### 功能详情

| 功能   | 说明                  | 优先级 |
| ---- | ------------------- | --- |
| 导入笔记 | 支持拖拽上传，自动解析内容       | P0  |
| 笔记列表 | 分页展示，支持筛选和排序        | P0  |
| 笔记详情 | 查看笔记内容，支持Markdown渲染 | P0  |
| 编辑笔记 | 修改笔记标题和内容           | P1  |

| 笔记分类 | 按文件夹/标签分类管理         | P2  |

```

```

### 3.4 风格学习模块

#### 功能列表

- 风格提取与分析
- 风格模板管理
- 风格预览
- 风格对比

#### 功能详情

| 功能     | 说明                 | 优先级 |
| ------ | ------------------ | --- |
| 自动风格提取 | 分析笔记文笔特征，生成风格向量    | P0  |
| 风格数据展示 | 可视化风格特征 (词汇/句式/结构) | P1  |
| 风格模板创建 | 手动定义风格模板           | P1  |
| 风格模板编辑 | 修改已有风格模板           | P2  |
| 风格对比   | 对比两种风格的差异          | P2  |

#### 风格特征维度

- **词汇特征**: 高频词汇、专有词汇、情感倾向
- **句式特征**: 平均句长、句式复杂度、标点使用
- **结构特征**: 段落组织、行文逻辑、开头结尾模式
- **写作习惯**: 人称使用、主动/被动语态、幽默感

## 4. Agent 功能说明

系统包含 5 个核心 AI Agent，通过协作完成文章生成任务。

### 4.1 Boss Agent

**职责**: 统筹系统任务调度和信息返回

**功能**:

- 接收用户请求，解析任务类型
- 协调其他Agent工作流程
- 汇总各Agent执行结果
- 返回统一响应格式
- 与 Gateway 层深度绑定

### 4.2 System Agent

**职责**: 系统内任务状态查询和参数返回

**功能**:

- 获取用户风格配置数据
- 获取系统运行参数
- 查询任务执行状态
- 返回配置类信息
- 与 Gateway 层深度绑定

### 4.3 Summary Agent

**职责**: 总结类工作

**功能**:

- 上下文压缩与总结
- 文章风格特征提取
- 长文本摘要生成
- 多文档融合总结

### 4.4 Writer Agent

**职责**: 文章撰写

**功能**:

- 根据主题和风格撰写文章
- 按照大纲逐段生成
- 调用工具完成特定内容
- 支持局部重写和润色

### 4.5 Review Agent

**职责**: 审查文章质量和是否违规

**功能**:

- 内容质量评分
- 敏感词和违规检测
- 生成修改建议
- 提供优化方向

## 5. 用户操作流程

### 5.1 首次使用流程

```
注册账号 → 导入笔记 → 系统自动学习风格 → 开始生成文章
```

### 5.2 文章生成完整流程

```
输入主题 → Gateway层调度 → BossAgent协调 → Agent协作生成 → ReviewAgent审核 → 人工确认 → 发布
```

### 5.3 风格学习流程

```
上传笔记 → 笔记清洗 → 风格分析 → 特征提取 → 风格模板生成 → 风格验证
```

### 5.4 Gateway 层工作流程

```
接收请求 → 任务创建 → BossAgent调度 → Agent执行 → 状态监控 → 结果返回
```

## 7. 非功能需求

### 7.1 性能需求

- 文章生成响应时间 < 60秒 (视长度而定)
- 知识检索响应时间 < 500ms
- 系统支持 100+ 并发用户
- Gateway 层 24 小时稳定运行

### 7.2 安全需求

- JWT令牌认证
- 密码加密存储 (bcrypt)
- API限流保护
- 敏感信息脱敏
- Agent调用权限控制

### 7.3 可用性需求

- 系统可用性 > 99.5%
- Gateway 层 24 小时运行
- 每日备份机制
- 异常告警通知
