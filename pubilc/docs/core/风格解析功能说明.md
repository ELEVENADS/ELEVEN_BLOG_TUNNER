# 风格解析功能说明

## 1. 功能概述

风格解析功能是 ELEVEN Blog Tuner 的核心能力之一，用于从文本中提取和分析写作风格特征。系统采用**混合模式**，结合规则统计、LLM 语义分析和 Embedding 向量三种技术，实现全面、准确的风格提取。

## 2. 架构设计

### 2.1 整体架构

```

           │    ┌───────────┘                │
```

## 3. 风格特征维度

### 3.1 统计特征（Statistical Features）

基于规则和统计计算的特征，具有确定性和可量化性。

#### 3.1.1 词汇特征

| 特征名 | 类型 | 说明 | 计算方式 |
|--------|------|------|----------|
| vocabulary_diversity | float | 词汇多样性 (0-1) | 唯一词数 / 总词数 |
| average_word_length | float | 平均词长 | 总字符数 / 总词数 |
| unique_words_ratio | float | 独特词比例 (0-1) | 唯一词数 / 总词数 |

#### 3.1.2 句式特征

| 特征名 | 类型 | 说明 | 计算方式 |
|--------|------|------|----------|
| average_sentence_length | float | 平均句长 | 总字符数 / 句子数 |
| sentence_complexity | float | 句式复杂度 | 从句标记数 / 句子数 |
| punctuation_density | float | 标点密度 | 标点符号数 / 总字符数 |

#### 3.1.3 结构特征

| 特征名 | 类型 | 说明 | 计算方式 |
|--------|------|------|----------|
| paragraph_average_length | float | 平均段落长度 | 总字符数 / 段落数 |
| transition_words_ratio | float | 过渡词比例 | 过渡词数 / 总词数 |

#### 3.1.4 写作习惯

| 特征名 | 类型 | 说明 | 计算方式 |
|--------|------|------|----------|
| passive_voice_ratio | float | 被动语态比例 | 被动句数 / 总句数 |
| first_person_ratio | float | 第一人称比例 | 第一人称词数 / 总词数 |
| emoji_usage | float | 表情符号使用率 | 表情符号数 / 总字符数 |

### 3.2 语义特征（Semantic Features）

基于 LLM 深度分析的特征，具有语义理解能力。

| 特征名 | 类型 | 说明 | 示例值 |
|--------|------|------|--------|
| language_style | string | 语言风格 | 幽默、严肃、活泼、沉稳、犀利、温和、文艺、朴实 |
| tone | string | 语气特征 | 正式、非正式、学术、口语化、亲切、客观 |
| vocabulary_level | string | 词汇水平 | 通俗、专业、学术、文学、口语、技术 |
| sentence_rhythm | string | 句式节奏 | 长短句交替、短句为主、长句为主、节奏明快 |
| rhetoric_devices | array | 修辞手法 | ["比喻", "排比", "设问", "反问", "夸张"] |
| emotional_tendency | string | 情感倾向 | 积极、消极、中性、热情、冷静、讽刺 |
| perspective | string | 叙述视角 | 第一人称、第三人称、客观、主观、全局 |
| logic_structure | string | 逻辑结构 | 总分总、递进、对比、并列、因果 |
| unique_habits | array | 独特表达习惯 | ["喜欢使用\"其实\"开头", "频繁使用破折号"] |
| target_audience | string | 目标读者群体 | 专业人士、大众、学生、技术从业者 |
| domain_characteristics | string | 领域特征 | 技术、文学、商业、学术、生活 |
| writing_quirks | string | 其他显著特点 | 自由描述 |

### 3.3 融合向量（Fusion Vector）

融合向量由三部分组成：

```
融合向量 = [文本语义向量] + [统计特征向量] + [语义特征编码]

- 文本语义向量：1536 维（OpenAI Embedding）
- 统计特征向量：11 维
- 语义特征编码：50 维（One-hot 编码）
- 总计：约 1597 维
```

## 4. API 接口

### 4.1 从文本学习风格

**POST** `/api/v1/styles/learn`

#### 请求参数

```json
{
  "text": "这是示例文本内容...",
  "style_name": "my_style",
  "metadata": {
    "source": "user_note"
  },
  "use_statistical": true,
  "use_semantic": true,
  "use_embedding": true
}
```

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| text | string | 是 | 文本内容 |
| style_name | string | 是 | 风格名称 |
| metadata | object | 否 | 元数据 |
| use_statistical | boolean | 否 | 是否使用规则统计特征，默认 true |
| use_semantic | boolean | 否 | 是否使用 LLM 语义分析，默认 true |
| use_embedding | boolean | 否 | 是否使用 Embedding 向量，默认 true |

#### 响应示例

```json
{
  "code": "200",
  "data": {
    "name": "my_style",
    "features": {
      "statistical": {
        "vocabulary_diversity": 0.75,
        "average_word_length": 4.2,
        "unique_words_ratio": 0.65,

        "sentence_complexity": 1.8,

        "paragraph_average_length": 150.5,
        "transition_words_ratio": 0.08,
        "passive_voice_ratio": 0.15,
        "first_person_ratio": 0.20,
        "emoji_usage": 0.02
      },
      "semantic": {
        "language_style": "幽默",
        "tone": "非正式",
        "vocabulary_level": "通俗",
        "sentence_rhythm": "长短句交替",
        "rhetoric_devices": ["比喻", "夸张"],
        "emotional_tendency": "积极",
        "perspective": "第一人称",
        "logic_structure": "递进",
        "unique_habits": ["喜欢使用\"其实\"开头"],
        "target_audience": "大众",
        "domain_characteristics": "生活",
        "writing_quirks": "经常使用网络流行语"
      }
```

## 5. 使用模式

### 5.1 快速模式（仅统计）

适用于大量文本快速处理，不调用 LLM。

```json
{
  "text": "...",
  "style_name": "quick_style",
  "use_statistical": true,
  "use_semantic": false,
  "use_embedding": true
}
```

**特点：**
- 速度快（毫秒级）
- 成本低（无 LLM 调用）
- 适合相似度比较

### 5.2 深度模式（全功能）

适用于需要深度风格分析的场景。

```json
{
  "text": "...",
  "style_name": "deep_style",
  "use_statistical": true,
  "use_semantic": true,
  "use_embedding": true
}
```

**特点：**
- 分析全面
- 包含语义理解
- 适合高质量风格模仿

### 5.3 轻量模式（仅语义）

适用于仅需 LLM 分析的场景。

```json
{
  "text": "...",
  "style_name": "semantic_style",
  "use_statistical": false,
  "use_semantic": true,
  "use_embedding": false
}
```

## 6. 增量学习

支持通过添加样本进行增量学习，系统会自动融合新旧特征。

**POST** `/api/v1/styles/{style_name}/add-sample`

```json
{
  "text": "新的写作样本..."
}
```

**融合策略：**
- 统计特征：加权平均
- 语义特征：保留最新
- 向量：加权平均

## 7. 风格相似度比较

使用余弦相似度比较两种风格的相似程度。

**GET** `/api/v1/styles/{style_name1}/compare/{style_name2}`

**响应：**
```json
{
  "code": "200",
  "data": {
    "similarity": 0.85
  },
  "message": "风格比较成功"
}
```

**相似度解释：**
- 0.9-1.0：极其相似
- 0.8-0.9：非常相似
- 0.7-0.8：比较相似
- 0.6-0.7：略有相似
- 0.6 以下：差异较大

## 8. 技术实现细节

### 8.1 SummaryAgent 风格分析

SummaryAgent 使用以下 Prompt 进行风格分析：

```
你是一位专业的写作风格分析专家。请深入分析用户提供的文本，提取其写作风格特征。

请从以下维度进行分析，并以 JSON 格式返回：
{
    "language_style": "语言风格标签...",
    "tone": "语气特征...",
    "vocabulary_level": "词汇水平...",
    ...
}
```

### 8.2 特征编码

语义特征使用 One-hot 编码转换为数值向量：

```python
def _encode_category(self, value: str, categories: List[str]) -> List[float]:
    vector = [0.0] * len(categories)
    if value:
        for i, cat in enumerate(categories):
            if cat in value:
                vector[i] = 1.0
                break
    return vector
```

### 8.3 延迟初始化

SummaryAgent 采用延迟初始化策略，避免不必要的资源消耗：

```python
def _get_summary_agent(self):
    if self._summary_agent is None and self.use_llm:
        from eleven_blog_tunner.agents.summary_agent import SummaryAgent
        self._summary_agent = SummaryAgent(llm_provider=self.llm_provider)
    return self._summary_agent
```

## 9. 使用建议

### 9.1 场景推荐

| 场景 | 推荐模式 | 说明 |
|------|----------|------|
| 快速预览 | 快速模式 | 毫秒级响应 |
| 风格对比 | 快速模式 | 统计特征足够 |
| 深度分析 | 深度模式 | 全面理解风格 |
| 文章生成 | 深度模式 | 高质量模仿 |
| 批量处理 | 快速模式 | 节省成本 |

### 9.2 最佳实践

1. **样本质量**：提供多样化的写作样本，至少 500 字以上
2. **样本数量**：3-5 个样本可获得较好的风格代表性
3. **定期更新**：随着写作风格演变，定期添加新样本
4. **命名规范**：使用有意义的风格名称，便于管理

## 10. 注意事项

1. **LLM 调用成本**：启用 `use_semantic` 会消耗 LLM API token
2. **文本长度限制**：LLM 分析限制前 3000 字符，超长文本会被截断
3. **异步处理**：风格学习是异步操作，大文本可能需要几秒完成
4. **存储空间**：风格向量约 1600 维，大量风格会占用较多存储
