# 短期记忆功能说明

## 1. 功能概述

短期记忆是 ELEVEN Blog Tuner 项目中的重要组件，用于管理对话历史和上下文信息，确保模型能够保持对话的连贯性和一致性。

## 2. 目录结构

```
eleven_blog_tunner/llm/
├── __init__.py              # 包初始化

├── factory.py               # LLM 工厂（支持记忆配置）
├── openai_provider.py       # OpenAI 提供商实现
├── local_provider.py        # 本地 Ollama 提供商实现
├── memory.py                # 记忆管理（核心实现）
└── test_memory.py           # 测试代码
```

## 3. 核心类设计

### 3.1 Memory 类

**功能**：管理单一会话的对话历史

**主要方法**：
- `add(role, content, **kwargs)`: 添加对话记录
- `get_history()`: 获取完整历史记录
- `get_context(max_tokens)`: 获取上下文（自动控制长度）
- `clear()`: 清空记忆
- `get_last_n(n)`: 获取最近 n 条记录
- `add_message(message)`: 添加完整消息对象
- `add_messages(messages)`: 添加多条消息
- `update_last_message(content, **kwargs)`: 更新最后一条消息

**特性**：
- 支持最大历史记录条数限制
- 支持基于 token 的上下文长度控制
- 自动管理上下文，确保不超过模型的 token 限制

### 3.2 SessionManager 类

**功能**：管理多个会话的记忆

**主要方法**：
- `get_session(session_id, **kwargs)`: 获取或创建会话
- `remove_session(session_id)`: 移除会话
- `list_sessions()`: 列出所有会话

**特性**：
- 支持多用户场景
- 会话隔离，避免不同用户的对话历史混淆

### 3.3 BaseLLM 集成

**功能**：将记忆系统集成到 LLM 基类

**主要方法**：
- `chat(messages, **kwargs)`: 对话（自动管理记忆）
- `stream_chat(messages, **kwargs)`: 流式对话（自动管理记忆）
- `get_memory()`: 获取记忆系统实例
- `clear_memory()`: 清空记忆
- `get_history()`: 获取对话历史
- `add_to_memory(message)`: 手动添加消息到记忆

**特性**：
- 自动合并历史消息
- 智能管理上下文长度
- 支持禁用记忆功能

## 4. 技术实现

### 4.1 上下文管理

```python
def _manage_context(self):
    """管理上下文长度，确保不超过token限制"""
    # 简单的token估算（实际应用中可能需要更精确的计算）
    total_tokens = 0
    temp_history = []
    
    # 从最新的消息开始，计算token数
    for message in reversed(self.history):
        content = message.get("content", "")
        # 粗略估算：每个字符约0.25个token
        message_tokens = len(content) * 0.25
        
        # 如果加上这条消息会超过限制，则停止
        if total_tokens + message_tokens > self.max_tokens:
            break
        
        temp_history.append(message)
        total_tokens += message_tokens
    
    # 重建历史，保留最近的消息
    if temp_history:
        self.history = deque(reversed(temp_history), maxlen=self.max_history)
```

### 4.2 LLM 集成

```python
async def chat(self, messages: List[Dict[str, Any]], **kwargs) -> str:
    """对话"""
    # 如果使用记忆系统，合并历史消息
    if self.use_memory:
        # 提取系统消息（通常是第一条）
        system_messages = [msg for msg in messages if msg.get("role") == "system"]
        # 提取新的用户消息
        new_messages = [msg for msg in messages if msg.get("role") != "system"]
        
        # 获取历史记忆（不包含系统消息）
        history = [msg for msg in self.memory.get_history() if msg.get("role") != "system"]
        
        # 构建完整消息列表：系统消息 + 历史消息 + 新消息
        full_messages = system_messages + history + new_messages
    else:
        full_messages = messages
    
    # 调用具体实现
    response = await self._chat(full_messages, **kwargs)
    
    # 更新记忆
    if self.use_memory:
        # 添加新消息到记忆
        for msg in new_messages:
            self.memory.add_message(msg)
        # 添加模型响应到记忆
        self.memory.add("assistant", response)
    
    return response
```

## 5. 使用方法

### 5.1 基本使用

```python
from eleven_blog_tunner.llm.memory import Memory

# 创建记忆实例
memory = Memory(max_history=10, max_tokens=4096)

# 添加对话记录
memory.add("user", "你好，你是谁？")
memory.add("assistant", "我是一个助手，有什么可以帮助你的吗？")

# 获取历史记录
history = memory.get_history()

# 获取上下文（自动控制长度）
context = memory.get_context(max_tokens=2048)

# 清空记忆
memory.clear()
```

### 5.2 会话管理

```python
from eleven_blog_tunner.llm.memory import SessionManager

# 创建会话管理器
manager = SessionManager()

# 获取或创建会话
session = manager.get_session("user123")

# 向会话添加消息
session.add("user", "我想订一张机票")
session.add("assistant", "好的，请问您要去哪里？")

# 列出所有会话
sessions = manager.list_sessions()

# 移除会话
manager.remove_session("user123")
```

### 5.3 与 LLM 集成

```python
from eleven_blog_tunner.llm.factory import LLMFactory

# 创建带记忆功能的 LLM 实例
llm = LLMFactory.create(
    provider="openai",
    model="gpt-4",
    use_memory=True,  # 启用记忆功能
    max_history=10    # 最大历史记录条数
)

# 第一次对话
messages = [
    {"role": "system", "content": "你是一个助手"},
    {"role": "user", "content": "你好，我叫张三"}
]
response = await llm.chat(messages)
print(response)

# 第二次对话，模型会记住你的名字
messages2 = [
    {"role": "user", "content": "你知道我叫什么名字吗？"}
]
response2 = await llm.chat(messages2)
print(response2)

# 查看对话历史
history = llm.get_history()

# 清空记忆
llm.clear_memory()
```

## 6. 配置选项

| 参数 | 描述 | 默认值 |
|------|------|--------|
| `use_memory` | 是否启用记忆功能 | `True` |
| `max_history` | 最大历史记录条数 | `10` |
| `max_tokens` | 最大上下文 token 数 | `4096` |

## 7. 最佳实践

1. **合理设置历史记录长度**：根据模型的上下文窗口大小设置合适的 `max_history` 值
2. **控制 token 限制**：根据模型的 token 限制设置 `max_tokens`，避免超出模型的上下文长度
3. **使用会话管理**：对于多用户场景，使用 `SessionManager` 管理不同用户的会话
4. **定期清理记忆**：对于长时间运行的应用，定期清理不需要的会话记忆
5. **结合长期记忆**：对于重要信息，考虑使用长期记忆（待实现）进行持久化存储
6. **记忆与提示词结合**：合理使用系统提示词和记忆功能，提高模型的响应质量
7. **监控记忆使用**：定期检查记忆使用情况，避免内存占用过高

## 9. 注意事项

1. **Token 估算**：当前使用简单的字符计数估算 token 数，实际应用中可能需要使用更精确的 token 计算方法
2. **性能考虑**：对于长对话，记忆系统可能会增加内存使用，需要合理设置历史记录长度
3. **上下文窗口**：不同模型的上下文窗口大小不同，需要根据模型类型调整 `max_tokens` 参数
4. **工具调用**：记忆系统会自动存储工具调用相关的消息，确保工具调用的连续性
5. **系统提示**：系统提示不会被存储到记忆中，每次对话都需要提供系统提示
