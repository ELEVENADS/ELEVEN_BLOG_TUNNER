# 模型工厂说明

## 1. 模块概述

LLM 模块是 ELEVEN Blog Tuner 项目的核心组件，负责与不同的大语言模型提供商进行交互。该模块采用工厂模式设计，支持多模型提供商的无缝切换，包括 OpenAI 和本地 Ollama 模型。

## 2. 目录结构

```
eleven_blog_tunner/llm/
├── __init__.py              # 包初始化
├── base.py                  # LLM 基类
├── factory.py               # LLM 工厂
├── openai_provider.py       # OpenAI 提供商实现
├── local_provider.py        # 本地 Ollama 提供商实现
└── memory.py                # 记忆管理
```

## 3. 核心类说明

### 3.1 BaseLLM（基类）

`BaseLLM` 是所有 LLM 提供商的抽象基类，定义了统一的接口和消息格式标准。

#### 标准消息格式

```python
[
    {"role": "system", "content": "你是一个助手"},
    {"role": "user", "content": "你好"},
    {"role": "assistant", "content": "你好！有什么可以帮助你的吗？"},
    {"role": "assistant", "content": "", "tool_calls": [...]},
    {"role": "tool", "content": "工具结果", "tool_call_id": "call_xxx", "name": "tool_name"}
]
```

#### 主要方法

- `chat(messages, **kwargs)`: 普通对话，返回完整响应
- `stream_chat(messages, **kwargs)`: 流式对话，返回响应片段
- `_prepare_messages(messages)`: 消息格式转换（子类重写）

### 3.2 LLMFactory（工厂类）

`LLMFactory` 负责创建和管理不同的 LLM 实例，支持根据配置或参数创建对应提供商的实例。

#### 支持的提供商

- `openai`: OpenAI 官方 API
- `local`: 本地 Ollama 模型

## 4. 使用方法

### 4.1 基本使用

#### 创建 LLM 实例

```python
from eleven_blog_tunner.llm.factory import LLMFactory

# 创建默认提供商实例（从配置文件读取）
llm = LLMFactory.create()

# 创建指定提供商实例
openai_llm = LLMFactory.create(provider="openai")
local_llm = LLMFactory.create(provider="local")

# 自定义模型参数
llm = LLMFactory.create(
    provider="openai",
    model="gpt-4-turbo",
    temperature=0.7,
    max_tokens=2048
)
```

#### 发送普通对话

```python
messages = [
    {"role": "system", "content": "你是一个助手"},
    {"role": "user", "content": "你好，你是谁？"}
]

response = await llm.chat(messages)
print(response)
```

#### 发送流式对话

```python
messages = [
    {"role": "system", "content": "你是一个助手"},
    {"role": "user", "content": "请介绍一下自己"}
]

async for chunk in llm.stream_chat(messages):
    print(chunk, end="")
```

### 4.2 工具调用

#### OpenAI 工具调用

```python
tools = [
    {
        "type": "function",
        "function": {
            "name": "get_weather",
            "description": "获取指定城市的天气",
            "parameters": {
                "type": "object",
                "properties": {
                    "city": {
                        "type": "string",
                        "description": "城市名称"
                    }
                },
                "required": ["city"]
            }
        }
    }
]

messages = [
    {"role": "system", "content": "你是一个助手，会使用工具回答问题"},
    {"role": "user", "content": "北京的天气怎么样？"}
]

response = await llm.chat(messages, tools=tools, tool_choice="auto")

# 处理工具调用
if isinstance(response, dict) and "tool_calls" in response:
    # 执行工具
    tool_result = await execute_tool(response["tool_calls"])
    
    # 继续对话
    messages.append({
        "role": "assistant",
        "content": "",
        "tool_calls": response["tool_calls"]
    })
    messages.append({
        "role": "tool",
        "content": tool_result,
        "tool_call_id": response["tool_calls"][0]["id"]
    })
    
    final_response = await llm.chat(messages)
    print(final_response)
```

#### Ollama 工具调用

Ollama 通过系统提示模拟工具调用，使用方式与 OpenAI 相同：

```python
# 与 OpenAI 相同的调用方式
response = await local_llm.chat(messages, tools=tools)
```

### 4.3 提供商切换

系统支持在不同提供商之间无缝切换，消息历史记录保持兼容：

```python
# 首先使用 OpenAI
openai_llm = LLMFactory.create(provider="openai")
messages = [
    {"role": "system", "content": "你是一个助手"},
    {"role": "user", "content": "你好"}
]
openai_response = await openai_llm.chat(messages)
messages.append({"role": "assistant", "content": openai_response})

# 切换到本地 Ollama
local_llm = LLMFactory.create(provider="local")
messages.append({"role": "user", "content": "请介绍一下自己"})
local_response = await local_llm.chat(messages)
print(local_response)
```

## 5. 配置管理

### 5.1 环境变量配置

在 `.env` 文件中配置 LLM 相关参数：

```env
# API 配置
API_KEY=your_openai_api_key

# LLM 配置
LLM_PROVIDER=openai  # 或 local
LLM_MODEL=gpt-4
LLM_TEMPERATURE=0.7
LLM_MAX_TOKENS=4096

# 本地 LLM 配置
LOCAL_LLM_BASE_URL=http://localhost:11434/api
LOCAL_LLM_MODEL=qwen3.5:9b
```

### 5.2 配置优先级

1. 函数调用时传入的参数（最高）
2. 环境变量
3. `.env` 文件
4. 默认值（最低）

## 6. 扩展指南

### 6.1 添加新的提供商

1. 创建新的提供商类，继承自 `BaseLLM`
2. 实现 `_chat` 和 `_stream_chat` 方法（注意：BaseLLM 已经实现了 `chat` 和 `stream_chat` 方法，子类需要实现 `_chat` 和 `_stream_chat`）

4. 在 `LLMFactory._providers` 中注册新提供商

        pass
    
    async def _chat(self, messages, **kwargs):
        # 实现对话逻辑
        pass
    
    async def _stream_chat(self, messages, **kwargs):
        # 实现流式对话逻辑
        pass

## 7. 记忆系统

### 7.1 概述

记忆系统是 ELEVEN Blog Tuner 的重要组件，用于管理对话历史和上下文，支持短期记忆和长期记忆（待实现）。

### 7.2 短期记忆

短期记忆通过 `Memory` 类实现，主要功能包括：

- 对话历史管理
- 自动上下文长度控制（基于 token 限制）
- 消息格式化和存储

### 7.3 会话管理

`SessionManager` 类用于管理多个会话，支持不同用户或场景的独立记忆。

### 7.4 使用方法

#### 基本使用

```python
from eleven_blog_tunner.llm.memory import Memory

# 创建记忆实例
memory = Memory(max_history=10, max_tokens=4096)

# 添加对话记录
memory.add("user", "你好，你是谁？")
memory.add("assistant", "我是一个助手，有什么可以帮助你的吗？")

# 获取历史记录
history = memory.get_history()

# 获取上下文（自动控制长度）
context = memory.get_context(max_tokens=2048)

# 清空记忆
memory.clear()
```

#### 会话管理

```python
from eleven_blog_tunner.llm.memory import SessionManager

# 创建会话管理器
manager = SessionManager()

# 获取或创建会话
session = manager.get_session("user123")

# 向会话添加消息
session.add("user", "我想订一张机票")
session.add("assistant", "好的，请问您要去哪里？")

# 列出所有会话
sessions = manager.list_sessions()

# 移除会话
manager.remove_session("user123")
```

#### 与 LLM 集成

```python
from eleven_blog_tunner.llm.factory import LLMFactory

# 创建带记忆功能的 LLM 实例
llm = LLMFactory.create(
    provider="openai",
    model="gpt-4",
    use_memory=True,  # 启用记忆功能
    max_history=10    # 最大历史记录条数
)

# 第一次对话
messages = [
    {"role": "system", "content": "你是一个助手"},
    {"role": "user", "content": "你好，我叫张三"}
]
response = await llm.chat(messages)
print(response)

# 第二次对话，模型会记住你的名字
messages2 = [
    {"role": "user", "content": "你知道我叫什么名字吗？"}
]
response2 = await llm.chat(messages2)
print(response2)

# 查看对话历史
history = llm.get_history()

# 清空记忆
llm.clear_memory()
```

### 7.5 配置选项

| 参数 | 描述 | 默认值 |
|------|------|--------|
| `use_memory` | 是否启用记忆功能 | `True` |
| `max_history` | 最大历史记录条数 | `10` |
| `max_tokens` | 最大上下文 token 数 | `4096` |

### 7.6 最佳实践

1. **合理设置历史记录长度**：根据模型的上下文窗口大小设置合适的 `max_history` 值
2. **控制 token 限制**：根据模型的 token 限制设置 `max_tokens`，避免超出模型的上下文长度
3. **使用会话管理**：对于多用户场景，使用 `SessionManager` 管理不同用户的会话
4. **定期清理记忆**：对于长时间运行的应用，定期清理不需要的会话记忆
5. **结合长期记忆**：对于重要信息，考虑使用长期记忆（待实现）进行持久化存储

## 9. 注意事项

1. **API 密钥**：使用 OpenAI 提供商时，需要在配置文件中设置有效的 API_KEY
2. **本地模型**：使用本地提供商时，需要确保 Ollama 服务正在运行（默认地址：http://localhost:11434）
3. **消息格式**：所有消息都应使用标准格式，系统会自动转换为各提供商的格式
4. **工具调用**：Ollama 的工具调用通过系统提示模拟，可能不如 OpenAI 原生支持稳定
5. **性能考虑**：本地模型的性能取决于硬件配置，可能不如云端模型响应迅速

## 10. 故障排除

### 10.1 OpenAI 错误

- **认证错误**：检查 API_KEY 是否正确
- **请求超时**：检查网络连接，或尝试减小 max_tokens
- **模型不存在**：确认模型名称是否正确，OpenAI 支持的模型列表可在官方文档中查看

### 10.2 Ollama 错误

- **404 错误**：确认 Ollama 服务是否正在运行
- **400 错误**：检查模型名称是否正确，或尝试重启 Ollama 服务
- **连接失败**：确认 Ollama 服务地址是否正确（默认：http://localhost:11434）

### 10.3 通用错误

- **消息格式错误**：确保使用标准消息格式
- **参数错误**：检查传入的参数是否符合各提供商的要求
- **依赖缺失**：确保已安装所有必要的依赖（`pip install -e .`）

## 11. 最佳实践

1. **统一消息格式**：始终使用标准消息格式，避免直接使用提供商特定的格式
2. **提供商抽象**：通过 LLMFactory 创建实例，便于后续切换提供商
3. **错误处理**：实现适当的错误处理，特别是网络请求和 API 调用
4. **参数调优**：根据具体任务调整 temperature、max_tokens 等参数
5. **工具调用**：对于复杂任务，使用工具调用来增强模型能力
6. **记忆管理**：合理使用记忆系统，平衡上下文长度和模型性能
7. **会话隔离**：对于多用户场景，使用会话管理器隔离不同用户的对话历史

通过以上方法，可以充分利用 LLM 模块的能力，实现与不同大语言模型的高效交互。
