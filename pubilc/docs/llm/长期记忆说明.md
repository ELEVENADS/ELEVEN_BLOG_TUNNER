# 长期记忆功能说明

## 1. 功能概述

长期记忆是 ELEVEN Blog Tuner 项目中的重要组件，用于存储和检索长期知识，基于向量数据库实现语义化的记忆管理。该功能支持本地和在线嵌入模型，为系统提供智能的知识存储和检索能力。

## 2. 技术架构

### 2.1 核心组件

- **LongTermMemory 类**：长期记忆的核心实现，集成 ChromaDB 向量数据库
- **嵌入模型**：支持本地 Ollama 模型和在线模型
- **向量数据库**：使用 ChromaDB 作为存储引擎

## 3. 核心实现

### 3.1 LongTermMemory 类

**功能**：管理长期记忆，基于向量数据库实现语义检索

**主要方法**：
- `__init__(vector_db_path, embedding_model, local_embedding_model, use_local_embedding)`: 初始化长期记忆
- `store(key, content, metadata)`: 存储记忆
- `retrieve(query, top_k)`: 检索相关记忆
- `delete(key)`: 删除记忆
- `get_stats()`: 获取统计信息

**特性**：
- 支持本地和在线嵌入模型
- 自动管理向量数据库
- 语义相似度检索
- 持久化存储

### 3.2 嵌入模型支持

**本地模型**：
- 模型名称：`dengcao/bge-large-zh-v1.5:latest`
- 基于 Ollama 运行
- 适合中文文本处理

**在线模型**：
- 默认模型：`text-embedding-3-small`
- 基于 OpenAI API
- 支持多语言

### 3.3 向量数据库

**ChromaDB**：
- 轻量级向量数据库
- 支持持久化存储
- 高性能检索
- 易于集成

## 4. 技术实现细节

### 4.1 向量数据库初始化

```python
def _init_chromadb(self):
    """初始化 ChromaDB"""
    try:
        import chromadb
        from chromadb.config import Settings
        
        # 初始化 ChromaDB 客户端
        self.client = chromadb.PersistentClient(
            path=self.vector_db_path,
            settings=Settings(
                chroma_db_impl="duckdb+parquet",
                persist_directory=self.vector_db_path
            )
        )
        
        # 创建或获取集合
        self.collection = self.client.get_or_create_collection(
            name="long_term_memory",
            metadata={"hnsw:space": "cosine"}
        )
    except ImportError:
        print("ChromaDB 未安装，请运行: pip install chromadb")
    except Exception as e:
        print(f"初始化 ChromaDB 失败: {e}")
```

### 4.2 嵌入向量生成

```python
async def _get_embedding(self, text: str) -> List[float]:
    """获取文本嵌入向量"""
    if self.use_local_embedding:
        return await self._get_local_embedding(text)
    else:
        return await self._get_online_embedding(text)
```

### 4.3 记忆存储

```python
async def store(self, key: str, content: str, metadata: dict = None):
    """存储长期记忆"""
    if not self.collection:
        print("向量数据库未初始化")
        return
    
    try:
        # 获取嵌入向量
        embedding = await self._get_embedding(content)
        if not embedding:
            print("获取嵌入向量失败")
            return
        
        # 存储到向量数据库
        self.collection.add(
            ids=[key],
            documents=[content],
            embeddings=[embedding],
            metadatas=[metadata] if metadata else None
        )
    except Exception as e:
        print(f"存储记忆失败: {e}")
```

### 4.4 记忆检索

```python
async def retrieve(self, query: str, top_k: int = 5) -> List[str]:
    """检索相关记忆"""
    if not self.collection:
        print("向量数据库未初始化")
        return []
    
    try:
        # 获取查询嵌入向量
        query_embedding = await self._get_embedding(query)
        if not query_embedding:
            print("获取查询嵌入向量失败")
            return []
        
        # 检索相关文档
        results = self.collection.query(
            query_embeddings=[query_embedding],
            n_results=top_k
        )
        
        # 返回检索到的文档
        return results.get("documents", [[]])[0]
    except Exception as e:
        print(f"检索记忆失败: {e}")
        return []
```

## 6. 使用方法

### 6.1 基本使用

```python
from eleven_blog_tunner.llm.memory import LongTermMemory

# 创建长期记忆实例
ltm = LongTermMemory(
    vector_db_path="./data/vector_db",
    use_local_embedding=True,
    local_embedding_model="dengcao/bge-large-zh-v1.5:latest"
)

# 存储记忆
await ltm.store("key1", "这是一条重要信息")

# 检索记忆
results = await ltm.retrieve("重要信息", top_k=3)

# 删除记忆
await ltm.delete("key1")

# 获取统计信息
stats = ltm.get_stats()
print(f"记忆数量: {stats.get('count')}")
```

### 6.2 与配置集成

```python
from eleven_blog_tunner.core.config import get_settings
from eleven_blog_tunner.llm.memory import LongTermMemory

settings = get_settings()

# 使用配置创建长期记忆实例
ltm = LongTermMemory(
    vector_db_path=settings.vector_db_path,
    embedding_model=settings.embedding_model,
    local_embedding_model=settings.local_embedding_model,
    use_local_embedding=settings.use_local_embedding
)

# 存储记忆
await ltm.store("article1", "这是一篇关于人工智能的文章...")

# 检索相关记忆
results = await ltm.retrieve("人工智能", top_k=5)
```

### 6.3 高级使用

```python
# 存储带元数据的记忆
metadata = {
    "type": "article",
    "author": "张三",
    "date": "2026-04-24"
}
await ltm.store("article2", "文章内容...", metadata=metadata)

# 批量存储记忆
memories = [
    ("mem1", "记忆1内容"),
    ("mem2", "记忆2内容"),
    ("mem3", "记忆3内容")
]
for key, content in memories:
    await ltm.store(key, content)

# 精确检索
results = await ltm.retrieve("具体查询内容", top_k=1)
```

## 9. 最佳实践

1. **选择合适的嵌入模型**：根据语言类型和性能要求选择本地或在线模型
2. **合理设置向量数据库路径**：选择合适的存储位置，确保有足够的磁盘空间
3. **优化检索参数**：根据实际需求调整 `top_k` 参数，平衡准确性和性能
4. **定期清理无用记忆**：及时删除不再需要的记忆，保持数据库大小合理
5. **监控记忆使用情况**：定期检查记忆数量和存储大小，避免过度增长
6. **结合短期记忆**：长期记忆用于存储重要信息，短期记忆用于对话上下文

## 10. 注意事项

1. **依赖安装**：使用前需要安装相关依赖，特别是 ChromaDB 和 Ollama
2. **本地模型要求**：使用本地嵌入模型需要 Ollama 服务正在运行，并且已下载 `dengcao/bge-large-zh-v1.5:latest` 模型
3. **在线模型要求**：使用在线嵌入模型需要有效的 API 密钥
4. **性能考虑**：嵌入向量生成和检索可能会消耗较多资源，特别是对于长文本
5. **存储限制**：向量数据库会占用磁盘空间，需要合理管理
6. **错误处理**：实现了基本的错误处理，但在生产环境中应添加更完善的异常处理
